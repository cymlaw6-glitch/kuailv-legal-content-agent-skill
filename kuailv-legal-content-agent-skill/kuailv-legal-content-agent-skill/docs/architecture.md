# 快律系统架构说明

## 1. 总体架构

「快律」采用多 Agent 编排架构，围绕法律内容生产流程构建七个主要 Agent：

1. 输入解析 Agent
2. RAG 法律研究 Agent
3. 法律推理 Agent
4. 内容转译 Agent
5. 短视频创作 Agent
6. 合规审查 Agent
7. 质量评估 Agent

系统输入为法律材料，输出为完整法律内容包。

## 2. 数据流

```text
原始材料
  ↓
结构化抽取
  ↓
法律依据检索
  ↓
法律关系推理
  ↓
通俗表达转译
  ↓
短视频内容生成
  ↓
合规审查与安全改写
  ↓
质量评分与多平台输出
```

## 3. RAG 设计

RAG 知识库可分为五类：

| 知识库 | 内容 |
|---|---|
| 法律法规库 | 民法典、刑法、劳动法、行政处罚法等 |
| 司法解释库 | 最高法、最高检相关司法解释 |
| 裁判规则库 | 指导性案例、典型案例、类案裁判规则 |
| 平台规则库 | 抖音、小红书、视频号等平台内容规范 |
| 内容模板库 | 高质量普法脚本、标题、封面文案模板 |

## 4. 关键技术点

- 长文本结构化抽取
- 法律问题识别
- RAG 检索与上下文拼接
- 多 Agent 状态传递
- 长链法律推理
- 法律内容通俗化改写
- 短视频传播结构生成
- 合规审查与安全改写
- 输出质量自动评分

## 5. 风险控制

系统默认采用保守合规策略：

- 不输出确定胜诉承诺；
- 不生成虚构法条；
- 不鼓励违法取证；
- 对具体案件保留不确定性；
- 对敏感事实进行隐私脱敏；
- 对高风险表达自动降级。
